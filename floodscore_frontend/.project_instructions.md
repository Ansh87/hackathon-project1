# FloodScore Frontend - Project Instructions

## Project Overview
Professional NextJS web application that integrates with a Flask API to provide climate risk assessment for homebuyers. The app analyzes flood and earthquake risks using FEMA and USGS data, calculates climate-adjusted property values for 2040, and identifies financial risks including insurance death spirals and market information asymmetries.

## Architecture & Tech Stack
- **Framework**: Next.js 14 with TypeScript
- **Styling**: Tailwind CSS
- **Charts**: Recharts (pie, line charts) + Plotly.js (gauge chart)
- **Icons**: Lucide-React
- **State Management**: React hooks (useState, useEffect)
- **Storage**: Browser localStorage for analysis history
- **API Client**: Fetch-based with error handling

## Key Components

### AnalysisForm (Client Component)
- Accepts: home_price, zip_code, flood_score_fema (0-100), earthquake_score_usgs (0-100)
- Validates all inputs before submission
- Calls API endpoint: POST http://127.0.0.1:5000/analyze
- Saves results to localStorage via saveAnalysis()
- Navigates to /results on success

### Results Page (Client Component)
- Displays all 6 API response fields with formatted values
- Shows 4 metric cards (value, loss %, asymmetry, death spiral)
- Bank insight in highlighted blue box
- Risk breakdown pie chart (flood vs earthquake %)
- Combined risk gauge (0-200 scale with color zones)
- 15-year value projection line chart

### AnalysisHistory (Client Component)
- Reads localStorage to display past 50 analyses
- Shows risk level badge, dates, key metrics per entry
- "View Results" button stores analysis to sessionStorage and navigates to /results
- "Delete" button removes from localStorage

## API Integration

### Environment Variables
- `NEXT_PUBLIC_API_URL`: Default is `http://127.0.0.1:5000` (set in .env)

### Request/Response Format
```typescript
// Request
{
  home_price: number;
  zip_code: string;
  flood_score_fema: number (0-100);
  earthquake_score_usgs: number (0-100);
}

// Response
{
  climate_adjusted_value_2040: number;
  annualized_loss_percent: number;
  insurance_death_spiral: boolean;
  information_asymmetry_gap: number;
  risk_level: string ('Extreme' | 'High' | 'Medium' | 'Low');
  bank_insight: string;
}
```

## Important Notes

1. **Client vs Server Components**:
   - Pages (home, analyze, results) are client components due to interactivity
   - Header is server component (static content)
   - All form handling and state in client components

2. **Hydration Safety**:
   - localStorage/sessionStorage access wrapped in `typeof window` checks
   - useEffect for client-side initialization
   - No non-deterministic values in SSR markup

3. **Type Safety**:
   - All data access uses optional chaining (?.)
   - Null/undefined guards with ?? fallbacks
   - @ts-ignore on Plotly.js dynamic import (known type issue)

4. **Charts**:
   - Recharts used for pie and line charts (native Next.js compatible)
   - Plotly.js for gauge (via dynamic import with ssr: false)
   - All charts sized to container with responsive height

5. **Forms & Validation**:
   - Zip code: exactly 5 digits
   - Home price: must be > 0
   - Both risk scores: required
   - Error messages shown inline

6. **History Management**:
   - Stored in localStorage under key 'floodscore_analyses'
   - Max 50 most recent analyses kept
   - sessionStorage used for temp data during results navigation
